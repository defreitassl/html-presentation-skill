# Optional Component Library

Use these components to diversify presentations without adding decorative filler. Add a component only when the source content supports it.

## Planning Components

- **Interactive calendar**: Use when source has dates, events, launches, cohorts, editorial schedules, releases, or workshops.
- **Milestone roadmap**: Use for phases, dependencies, gates, rollout plans, implementation sequences, or decision timelines.
- **Workstream filter**: Use when repeated items belong to teams, tracks, product areas, markets, cohorts, channels, or priority groups.
- **Owner matrix**: Use for accountability, RACI-like detail, escalation paths, approvals, or operating routines.
- **Readiness checklist**: Use for launch, training, migration, validation, release, or meeting prep.

## Executive Components

- **Decision register**: Use when the reader must approve, defer, fund, assign, or choose.
- **Scenario matrix**: Use when outcomes vary by risk, investment level, adoption, staffing, policy, or market response.
- **Risk accordion**: Use for important but secondary risk detail.
- **Recommendation stack**: Use when the source has one preferred path and alternatives.
- **Evidence table**: Use when claims need traceable facts, source notes, assumptions, or confidence levels.

## Technical Components

- **System map**: Use when the source includes services, integrations, environments, teams, queues, data flows, or dependencies.
- **Status board**: Use for health, severity, priority, SLA, readiness, incidents, or progress.
- **Tabbed runbook**: Use for setup, operate, recover, validate, monitor, or migrate modes.
- **Validation matrix**: Use for test criteria, observability signals, controls, guardrails, or acceptance gates.
- **Incident timeline**: Use for postmortems, migrations, launches, failure recovery, or staged cutovers.

## Textual Document Components

- **Document cover**: Use for memos, reports, policies, white papers, legalistic sources, and executive narratives.
- **Table of contents**: Use for long documents and async reading.
- **Pull quote**: Use for the central thesis, policy principle, or executive takeaway.
- **Margin note**: Use for caveats, assumptions, source notes, definitions, or pending decisions.
- **Evidence callout**: Use to anchor a claim to facts without turning the section into a dashboard.
- **Appendix accordion**: Use for detailed source material that should remain available but not dominate the main read.

## Adaptation Rules

- Remove components that do not have source-backed content.
- Combine similar components instead of repeating the same idea visually.
- Prefer one strong custom composition over many weak decorative blocks.
- Keep interaction state accessible with `aria-expanded`, `aria-selected`, `aria-pressed`, or `aria-controls`.
- Do not hide the main conclusion inside tabs, filters, accordions, or appendices.

# Style Presets

Offer these presets when the user has not specified a visual direction. Keep the question short:

```text
Which visual style should I use: executive briefing, editorial report, technical command center, community platform, product launch, training field guide, data dashboard, or minimal board memo?
```

If the user does not choose, select the preset that best matches the audience and source material.

## Executive Briefing

Use for leadership, board, sponsor, investor, or public-sector decision presentations.

- Tone: sober, clear, high trust.
- Layout: dark hero, sidebar navigation, summary cards, operating map, roadmap, risk section, decision close.
- Color: deep neutral base with 2-3 confident accents.
- Interactions: accordions for risks, tabs for scope, progress bar.

## Editorial Report

Use for research, strategy, policy, annual reports, and narrative-heavy content.

- Tone: magazine-like, refined, text-forward.
- Layout: strong typography, section bands, pull quotes, evidence tables, chapter navigation.
- Color: paper-like background with high-contrast ink and a restrained accent.
- Interactions: expandable source notes and section navigation.

## Technical Command Center

Use for architecture, security, platform, infrastructure, DevOps, incident, or engineering plans.

- Tone: precise, operational, systems-oriented.
- Layout: console/dashboard hero, architecture map, dependency grid, runbook steps, validation checklist.
- Color: dark shell with high-contrast status colors.
- Interactions: tabs, status filters, detail panels.

## Community Platform

Use for onboarding, member journeys, Discord/Slack/Circle communities, support operations, education cohorts, or volunteer programs.

- Tone: welcoming, organized, active.
- Layout: community preview hero, onboarding flow, role cards, channel map, governance accordions, launch checklist.
- Color: approachable neutrals with lively but controlled accents.
- Interactions: role detail panel, accordions, channel/category filters.

## Product Launch

Use for product announcements, feature launches, GTM plans, demos, or sales enablement.

- Tone: confident, polished, momentum-driven.
- Layout: product hero, audience/problem framing, feature cards, rollout timeline, metrics, launch checklist.
- Color: brand-forward with a strong primary accent.
- Interactions: feature tabs, launch-phase timeline, pricing/package comparisons when relevant.

## Training Field Guide

Use for tutorials, courses, internal enablement, SOPs, workshops, and onboarding documents.

- Tone: practical, instructional, step-by-step.
- Layout: learning objective, sequence map, examples, checkpoints, FAQ, reference section.
- Color: calm, readable, with clear callout colors.
- Interactions: accordions, checklists, step navigation.

## Data Dashboard

Use for analytics, metrics reviews, performance reports, OKRs, and monitoring summaries.

- Tone: analytical, focused, evidence-first.
- Layout: KPI hero, trend cards, comparison tables, findings, risks, recommendations.
- Color: neutral background with status colors.
- Interactions: filters, tabs for cohorts/timeframes, expandable metric definitions.

## Minimal Board Memo

Use when the user wants a restrained, fast-loading, low-decoration artifact.

- Tone: concise, formal, decision-oriented.
- Layout: title page, memo sections, tables, recommendation, next steps.
- Color: mostly neutral with one accent.
- Interactions: minimal; use navigation and accordions only when useful.
